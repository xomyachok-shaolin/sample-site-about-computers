Задание и его выполнение. ЛР2.

Часть 1.

  1. Изучить назначение и ключи команды ln.
    ln --help

    - создать жесткую ссылку на файл.
    ln asminaev/file file

    Просмотреть содержимое файла, используя ссылку.
    cat file

    Удалить файл.
    rm -i asminaev/file

    Просмотреть содержимое файла.
    cat file

    Объяснить результат;
    При удалении имени удаляется только связь,
    если связей больше нет удаляется весь файл.

    - создать жесткую ссылку на каталог.
    ln asminaev/Загрузки Загрузки

    Объяснить результат;
    не допускается создавать жёсткие ссылки на каталоги


  2. Выполнить все задания пункта 1, создавая не жесткие, а символьные ссылки.
    ln -s asminaev/file file
    cat file
    rm -i asminaev/file
    Если остались символические ссылки, доступ к файлу по ним будет невозможен.
    ln -s asminaev/Загрузки Загрузки
    В отличие от прямых ссылок, символические ссылки можно использовать для создания ссылок на каталоги.

  3. Создать жесткую и символьную ссылки на файл.
    ln -s easyfile s_file; ln hardfile h_file

    С помощью команды ls просмотреть inod файла и ссылок.
    ls -l

    Объяснить результат.
    Символические ссылки имеют собственный тип, обозначенный как l. Тип файла для s_file - l, т.е. это символическая ссылка, а не обычный файл. Число, стоящее после group - это размер файла. Обратите внимание: размеры разные. Размер файла s_file составляет меньшее кол-во байт. Это обусловлено тем, что s_file - всего лишь символическая ссылка, хранящая путевое имя реально существующего файла,занимающее всего несколько байтов. Индексные дескрипторы двух имен hardfile и h_file файлов совпадают, это значит, что они относятся к одному и тому же файлу.

Часть 2.

  1. Изучите при помощи man опцию -l команды ls.
    man ls

    Просмотрите права каталогов /etc, /bin и домашнего каталога.
    ls -ld /etc /bin

    Просмотрите права файлов, содержащиеся в этих каталогов.
    ls -l /etc; ls -l /bin

    Выявите тенденции (файлов с какими правами в каких каталогах больше).
    в каталоге bin только симв ссылка

  2. Изучите материал, посвящѐнный пользователям и группам пользователей.
    man users; man groups

    Изучите руководство по командам chown и chgrp.
    man chown; man chgrp

    Выясните, кто является владельцем и к какой группе владельцов принадлежат файлы вашего домашнего каталога, каталогов /etc, /root, /bin и /dev.
    ls -l /etc /root /bin /dev

  3. Определите атрибуты файлов /etc/shadow и /etc/passwd попробуйте вывести на экран содержимое этих файлов.
    ls -l /etc/shadow /etc/passwd

  4. Изучите команду chmod.
    man chmod

    Создайте в домашнем каталоге любые четыре файла, установите при помощи восмеричных масок на каждый из них в отдельности следующие права:
    - для себя все права, для группы и остальных - никаких;
    chmod 770 file1

    - для себя чтение и запись, для группы чтение, для остальных - все;
    chmod 647 file2

    - для себя исполнение и запись, для группы никаких, для остальных чтение;
    chmod 304 file3

    - для себя запись, для группы все, для остальных - только запись.
    chmod 272 file4

  5. Выполните задание предыдущего пункта, используя в команде chmod только символы прав доступа.
    chmod u=rwx file1;chmod u=rw g=r o=rwx file2;chmod u=wx o=r file3;chmod u=x g=rwx o=w file4

  6. В домашнем каталоге создайте файл и установите на него права так, чтобы его можно было только редактировать.
    chmod 222 file

  7. Скопируйте в свой домашний каталог файл ls из каталога /bin.
    cp -i /bin/ls

    Запретите выполнение этого файла и попробуйте выполнить именно его, а не исходный(!).
    chmod 000 ls

  8. Попробуйте зайти в каталог /root, объясните результат и причину.
    cd /root ~ Переносит вас в домашний каталог пользователя root
